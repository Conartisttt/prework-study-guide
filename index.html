HTML

  • HTML stands for Hypertext Markup Language
  • HTML elements work together to build the structure of a webpage, and provide a framework that makes it easy for users to navigate the webpage.
    • This can be text, images, or parts of a page's layout
  • The head element contains information about the webpage. Elements included within the head element are not visible when a user looks at a webpage. It is metadata.
  • Metadata is data about our webpage that the broswer uses to effectively serve our webpage to users.
  • Inside the head elements, we can use the following metadata tags:
    • title: defines the title for the webpage. This element is required and is extremely important for Search Engine Optimiziation.
    • style: defines the CSS styles associated with a website.
    • base: defines the base URL for a webpage.
    • link: connects an external resource to the HTML document.
    • meta: defines metadata such as the character set, description, keywords, author, and viewport.
    • script: embeds client-side scripts in the HTML document.
  • The body elements represents the visible content shown to the user.

CSS

  • CSS stands for Cascading Style Sheets.
  • This language is used to describe the presentaiton of a website that's created with HTML, and how each element on the page is rendered
    • This might include fonts, colors, and spacing
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • class attribute can be assigned to any viewable HTML element. It is used to assign a CSS class to items and can be used for mulitiple items.
    • class is represented by a . in CSS
  • id attribute can be assigned to only one single viewable HTML element. It's value is always unique per webpage and cannot be usued for multiple items.
    • id is represented by a # is CSS
  • universal attribute can be assigned to the entire documents HTML.
    • universal is represented by a * in CSS
  • span is an inline container that is sued to mark up part of a text, or part of a document.

Git

  • Git is software on your computer that tracks changes that you make to your code.
  • GitHub is a cloud-based service that allows developers to store, manage, and collaborate on code inside repositories. It is connected to Git.
  • Repositories, or repos: used as folders to store the following:
    • Files that contain code.
    • Files that contain other information about the code, like a README file.
    • The version history for all of the files in the repository.
  • Git commands: always start with git, and are used to save and track versions of your code. Using git commands, we can initaiate or create a repo, add our code to the repo, and view the different versions of our code, which are each stored in a unique commit.
    • git revert command, followed by the SHA, is used to revert a project back to an older version. This is often used after discovering a bug in your code.
    • git status command checks what branch we are currently on
    • git checkout -b branch-name command creates a new branch and switches to it
    • After making changes to a branch in VS Code, you need to sync your working branch back to the main branch. This is called the git flow.
    • Performing the Git flow:
      • git add -A command to stage a commit for all files in the working branch.
      • git commit -m "insert description here" command commits any changes with a short descriptive message of the changes implemented.
      • git pull origin main command pulls the base branch into our feature branch so that it is syncing with the most up to date version in the repo
      • git push origin "branch-name" command pushes all of our changes up to the repository on GitHub to be pulled in
      • Navigate to your repository on GitHub. Click "Pull Requests" tab and then the "New pull request" button.
      • Make sure the base branch is main and the comparison branch is the feature branch you just updated. Add a title for the pull request along with a short description of the changes.
      • Click the "Create pull request" button.
      • Now we can merge the comparison branch into the base branch by selecting the "Merge pull request" button and then confirming the merge request.
      • If you had an issue open on GitHub for this change, you can now navigate to this issue, and close it.
      • Now, make sure you update the local main branch. Back in the command line, navigate to the main branch by entering: git checkout main
      • Then pull the remote main branch into the local main branch by entering: git pull origin main

JS

  • JS stands for JavaScript.
  • This language is used to provide logic and interactivity for websites.
  • Variable: a named container that allows us to store data in our code.
  • Control flow: the order in which a computer executes code in a script.
  • Conditional logic: a simple way to tell a computer to perform an action upon a certain condition being met.
  • Array: a single variable that is used to hold a group of related data.
  • Index: a unique number given to each individual data inside an array
  • Loops: a sequence of instructions that is continually repeated until a certain condition is reached.
  • Function: simply a set of instructions that tells the computer how to perform a task. A function must be called on if we want it to execute.
  • Use /* java script code is here */ to comment out code. This makes it inactive without deleting it, in case we need it again later.

Other Concepts & Terminology

  • User story: a short description of the user you're creating an application for. It consists of the following three sections:
    • The type of person using your application.
    • What the user wants from the application.
    • Why the user wants what they want. What problem are they trying to solve?
  • Acceptance criteria: a list of the things that your application must acheive in order to be considred done.
  • Computational thinking: the process of understanding a complex problem by breaking that problem into smaller parts, and then developing possible solutions that can be clearly presented in a way that comupters and/or humans can understand. There are four cornerstones:
    • Decomposition: breaking down the problem into smaller tasks
    • Pattern recognition: thinking about how we've solved subtasks previously, and finding any patterns that might help us to solve this particular problem.
    • Abstraction: make sure that we are only focusing on relevant information, and that we are disregarding details that won't help us solve this problem.
    • Testing and debugging algorithms: an algorithm is a sequence of steps or rules that we can use to solve our problem. Changing our original algorithm after finding a mistake or a potenial mistake in our code is called debugging.
  • Event trigger: an association between a predefined event, and the script that is to be executed when that event occurs.
  • Conditional statement: allow us to make decisions based on conditions in our code.
  • For loop: an action that will repeat over and over again until a condition is met.
  • Agile software development, AKA agile: a framework that helps developers handle change and uncertainty during the development process.
  • Command line interface, or CLI: a command-line program that allows us to input text commands to execute operating system functions, like opening a file or creating a new folder. Here are a commands to remember:
    • ls: allows uss to see a list of files and directories on our computer, or in the current directory we are in.
    • cd: "change directories" will allow us to change directories. Type the directory you would like to change to after using this command, and press enter.
    • cd..: allows us to navigate backwards one directory at a time.
    • pwd: "print working directory" will print out the directory that we are currently in to the terminal.
    • mkdir: creates a directory. Type the name of the directory you would like to create after this command, and press enter.
    • touch: creates a new file. Type the name of the file you would like to create.type of file, and press enter.
    • open: opens a file. Type the name of the file you would like to open after this command, and press enter.
  • Graphical user interface, or GUI: includes icons and menus that allow users to navigate their system by using a mouse, touchpad, or assistive technology.
  • Terminal: the application that allows us to interact with the shell.
  • Shell: the software that interprets and executes the various commands that we type in the terminal.
  • Bash, or Bash commands: this term is used to describe the shell and the types of commands that we will use. Bash is just a popular version of a shell that will allow us to perform many tasks from the command line inside the terminal.
  • Homebrew: a package manager that makes it easy to install and update applications on macOS.
  • Package: contains code written by others that we can add to and use on our computers and in our projects.
  • Package manager: automates the process of isntalling, updating, configuring, and removing packages.
  • Pair programming: occurs when two developers collaborate on a feature or a project.
  • Driver-navigator model: one of the most common pair programming strategies. One developer uses a keyboard to type the code, known as the driver. The other developer is responsible for the overall direction that the pair decides to go, known as the navigator.
  • SSH key: an access credential for the secure shell, SSH, network protocol. Generating SSH keys allows developers to interface with certain remote services without having to constantly retype login information.
  • SSH agent: a program that keeps track of a user's identity keys and passphrases.
  • Web development project: a web application, or web app, that provides a solution to a real-world problem.
  • Minimum viable product, or MVP: this is usually the first iteration of a product without extra features. It is the first version that fulfills the requirements and proves that the solution to the program works.
  • Version control systems: allow us to track and manage changes to our codebase. Git is the most commonly used version control software among developers.
  • SHA, or hash: a unique identifier that allows us to keep a record of the specific changes committed to version control.
    • SHA is actually a long string of numbers, or letters. Developers often type out only the first seven numbers or letters.
  • Control flow: the order in which a computer executes code in a script.
