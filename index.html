HTML

  • HTML stands for Hypertext Markup Language
  • HTML elements work together to build the structure of a webpage, and provide a framework that makes it easy for users to navigate the webpage.
    • This can be text, images, or parts of a page's layout
  • The head element contains information about the webpage. Elements included within the head element are not visible when a user looks at a webpage. It is metadata.
  • Metadata is data about our webpage that the broswer uses to effectively serve our webpage to users.
  • Inside the head elements, we can use the following metadata tags:
    • title: defines the title for the webpage. This element is required and is extremely important for Search Engine Optimiziation.
    • style: defines the CSS styles associated with a website.
    • base: defines the base URL for a webpage.
    • link: connects an external resource to the HTML document.
    • meta: defines metadata such as the character set, description, keywords, author, and viewport.
    • script: embeds client-side scripts in the HTML document.
  • The body elements represents the visible content shown to the user.

CSS

  • CSS stands for Cascading Style Sheets.
  • This language is used to describe the presentaiton of a website that's created with HTML, and how each element on the page is rendered
    • This might include fonts, colors, and spacing
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

Git

  • Git is a free, open source version control system that allows web developers to collaborate on projects without overwriting each other's work. It also allows developers to rever to previous verions of the code if a problem is discovered.
  • GitHub is the largest provider of internet hosting for software and web development code, and provides the version control functionality of Git in a central repository
  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it

Other Concepts & Terminology

  • User story: a short description of the user you're creating an application for. It consists of the following three sections:
    • The type of person using your application.
    • What the user wants from the application.
    • Why the user wants what they want. What problem are they trying to solve?
  • Acceptance criteria: a list of the things that your application must acheive in order to be considred done.
  • Computational thinking: the process of understanding a complex problem by breaking that problem into smaller parts, and then developing possible solutions that can be clearly presented in a way that comupters and/or humans can understand. There are four cornerstones:
    • Decomposition: breaking down the problem into smaller tasks
    • Pattern recognition: thinking about how we've solved subtasks previously, and finding any patterns that might help us to solve this particular problem.
    • Abstraction: make sure that we are only focusing on relevant information, and that we are disregarding details that won't help us solve this problem.
    • Testing and debugging algorithms: an algorithm is a sequence of steps or rules that we can use to solve our problem. Changing our original algorithm after finding a mistake or a potenial mistake in our code is called debugging.
  • Event trigger: an association between a predefined event, and the script that is to be executed when that event occurs.
  • Conditional statement: allow us to make decisions based on conditions in our code.
  • For loop: an action that will repeat over and over again until a condition is met.
  • Agile software development, AKA agile: a framework that helps developers handle change and uncertainty during the development process.
  • Command line interface, or CLI: a command-line program that allows us to input text commands to execute operating system functions, like opening a file or creating a new folder.
  • Graphical user interface, or GUI: includes icons and menus that allow users to navigate their system by using a mouse, touchpad, or assistive technology.
  • Terminal: the application that allows us to interact with the shell.
  • Shell: the software that interprets and executes the various commands that we type in the terminal.
  • Bash, or Bash commands: this term is used to describe the shell and the types of commands that we will use. Bash is just a popular version of a shell that will allow us to perform many tasks from the command line inside the terminal.
  • Homebrew: a package manager that makes it easy to install and update applications on macOS.
  • Package: contains code written by others that we can add to and use on our computers and in our projects.
  • Package manager: automates the process of isntalling, updating, configuring, and removing packages.
